# User table:
## Описание таблицы
Таблица user - содержит данные о пользователе, а так же данные, указанные при регистрации и нужные для аутентификации.
## Функциональные зависимости
{user_id} -> user_name, nick_name, email, password, birth_time, gender, avatar_url, creation_time, update_time\
{nick_name} -> user_id, user_name, email, password, birth_time, gender, avatar_url, creation_time, update_time\
{email} -> user_id, user_name, nick_name, password, birth_time, gender, avatar_url, creation_time, update_time

- **user_id** - первичный ключ
- **nick_name** - альтернативный ключ
- **email** - альтернативный ключ

## Нормальная форма
Таблица находится в первой нормальной форме, т.к. все атрибуты простые и не повторяются.\
Таблица находится во второй нормальной форме, т.к. нет функциональной зависимости от части первичного ключа. Нет частичных функциональных зависимостей, каждый атрибут функционально полно зависит от первичного ключа.\
Таблица находится в третьей нормальной форме, т.к. все атрибуты нетранзитивно зависят от первичного ключа.\
Таблица находится в НФБК, т.к. существуют только такие функциональные зависимости, у которых детерминант первичный ключ.
